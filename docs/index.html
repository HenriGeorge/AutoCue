      How to use AutoCue (web mode)

      1. Export from Rekordbox: File → Export Collection in rekordbox format → save as rekordbox.xml.
      2. Drop the XML above. Optionally also drop your audio files (or the whole analysis folder) to enable playback, artwork, and phrase analysis.
      3. Choose analysis mode:
        • Bar intervals — places cues every N bars using the BPM grid. Works for every analyzed track.
        • ✨ Phrase analysis — uses phrase data from the dropped ANLZ files (Intro, Verse, Chorus, Outro). Drop your Rekordbox analysis folder to enable this.
      4. Adjust settings: bars interval, starting bar, max hot cues per track, and optional memory cue at bar 1. Hover any option for a tooltip explaining what it does.
      5. Preview cues: click any cue badge on a track card to audition the position. The timeline at the bottom of each card shows all cue positions at a glance.
      6. Download the updated XML, then import it back into Rekordbox: File → Import Library → select the file.

      Everything runs locally in your browser — no files are ever uploaded anywhere.

      Local server mode — workflow

      1. Close Rekordbox before applying, deleting, or coloring tracks. AutoCue writes directly to the Rekordbox database (master.db) — it must be closed first. AutoCue warns you if it detects Rekordbox is still running.
      2. Filter & browse your library:
        • Use the search bar to filter by title or artist.
        • Use the playlist dropdown to view a specific playlist.
        • Filter by Phrase only, Rating, Plays, Last Played, or My Tag.
        • Sort by Title, Artist, Album, BPM, Key, Rating, or Plays — click the same button again to reverse order.
        • In Album view tracks are grouped under collapsible album headers.
      3. Choose analysis mode:
        • Bar intervals — places cues every N bars using the BPM grid. Works for every analyzed track.
        • ✨ Phrase analysis — uses Rekordbox's built-in phrase detection (Intro, Verse, Chorus, Outro, etc.). Falls back to bar intervals if a track has no phrase data.
      4. Configure cue settings (hover each option for a detailed tooltip):
        • Bars interval / Start bar / Max hot cues — control bar-interval placement.
        • Memory cue at bar 1 — also writes a CDJ Auto Cue so your CDJ loads to the right position.
        • Fill-beat cues — phrase mode only; adds cues at the fill sections before drops.
        • Skip tracks with existing cues — leaves manually-cued tracks untouched.
      5. Select tracks (optional): check individual track cards to scope operations to a subset. Select all selects every visible (filtered) track. Apply, Delete, and Color by BPM all respect the current selection.
      6. Preview cues (optional): click Preview cues to generate positions without writing. Each card shows a secondary preview timeline. Click Apply when ready.
      7. Apply to Rekordbox: click the green button. AutoCue backs up master.db first, then streams progress as each track is processed. Use ↩ Undo last apply immediately afterwards to restore the backup if needed.
      8. Color tracks by BPM: assigns a Rekordbox track color by BPM range — Aqua (<90), Green (90–114), Blue (115–124), Orange (125–134), Red (135–149), Pink (≥150). Enable Skip colored to leave already-colored tracks untouched.
      9. Delete all hot cues: click Delete all cues → confirm. Removes every hot cue (slots A–H) from all visible tracks, including manually placed ones. A backup is made first.
      10. Restore a backup: click Restore backup to pick from previous database snapshots saved to ~/.autocue/backups/.
      11. Listen to tracks: hover over the album art and click play. AutoCue loads the audio directly from disk — no file dropping needed. You can also drop an audio file onto the Load audio button on any card.

      Backups are saved to ~/.autocue/backups/ before every write operation. The server only accepts connections from localhost — your database never leaves your machine.

      Use the Python CLI or local server instead ↓

      The Python package provides both a CLI and a local server that writes directly to your Rekordbox database — no XML export/import needed.

      pip install autocue

# Local server (this UI) — reads & writes the Rekordbox DB directly
autocue serve                        # opens browser automatically
autocue serve --no-browser           # start without opening browser
autocue serve --port 8080            # use a different port

# CLI — generates a new rekordbox.xml with cues
autocue --library                    # all analyzed tracks
autocue --library --overwrite        # re-generate for all (overwrite existing)
autocue --library --dry-run          # preview without writing
autocue --track "Song Title"
autocue --track-id 42 --dry-run
autocue --library --playlist "My Set"
autocue --library --mode phrase      # force phrase mode
autocue --library --mode bar --bars-interval 8 --start-bar 1
