Part 1 · Setup on a new machine

AutoCue reads your friend's Rekordbox database directly and writes hot cues back into it — no XML export/import. It runs as a small local web server on their own computer. Nothing is uploaded anywhere.

Before you start — prerequisites

RequirementWhy / how
macOS (recommended)The Rekordbox database path is auto-detected on macOS. Windows works too but needs one extra flag (see troubleshooting).
Rekordbox 6 or 7, installed & opened at least onceAutoCue reads its library. The encryption key is taken automatically from the installed Rekordbox — so it must be a real Rekordbox install.
A library that has been analyzedIn Rekordbox, tracks must have BPM + (ideally) phrase analysis run, or there will be nothing to cue.
Python 3.10 or newerCheck with python3 --version. If missing, install from python.org or via Homebrew (brew install python).
⚠ Golden rule: quit Rekordbox before writing
Rekordbox locks its database while open. AutoCue can browse/analyze, but every write (applying cues, deleting duplicates, saving playlists) requires Rekordbox to be fully quit — the app refuses the write otherwise with a clear message.

Install — step by step

1
Open Terminal. (Applications → Utilities → Terminal, or ⌘ Space → "Terminal".)
2
Get the code. With git: 
git clone https://github.com/HenriGeorge/AutoCue.git
cd AutoCue
No git? Download the ZIP from the GitHub page (green Code button → Download ZIP), unzip it, then in Terminal cd into the unzipped AutoCue folder.
3
(Recommended) Create an isolated environment so AutoCue's packages don't touch the system Python: 
python3 -m venv .venv
source .venv/bin/activate
You'll see (.venv) at the start of the prompt. Re-run source .venv/bin/activate in any new Terminal before autocue.
4
Install AutoCue. This one command pulls everything it needs — including the SQLCipher reader for the encrypted Rekordbox DB — automatically: 
pip install -e .
Optional: add YouTube download support with pip install -e ".[download]" (also needs ffmpeg on the PATH — brew install ffmpeg).
5
Quit Rekordbox (⌘Q — fully quit, not just close the window).
6
Start AutoCue. 
autocue serve
It prints a line like Serving on http://localhost:7432 and opens that page in the default browser. That's the app. Leave the Terminal window open while you use it; press Ctrl C in Terminal to stop the server.
✓ That's the whole install
No Homebrew SQLCipher, no manual key download, no build step. pip install -e . brings in pyrekordbox (which bundles the database reader), fastapi, and uvicorn. The database key is read from the Rekordbox install automatically.

What AutoCue does to stay safe

Before any write to the Rekordbox database, AutoCue:

  • Confirms Rekordbox is not running — and aborts with a clear error if it is.
  • Creates a timestamped backup at ~/.autocue/backups/master_TIMESTAMP.db.
  • Writes inside a transaction savepoint that rolls back automatically on any failure.

You can roll back the most recent change from the app's Restore backup… button (top bar).

Troubleshooting

SymptomFix
autocue: command not foundThe environment isn't active. Run source .venv/bin/activate in the AutoCue folder, or re-run pip install -e ..
"Rekordbox is running — close it…" (409)Fully quit Rekordbox (⌘Q) and retry the write. Browsing/analysis works, but writes need it closed.
Database won't open / cipher / key errorMake sure Rekordbox 6/7 is actually installed and has been opened at least once on this Mac (that's where the key comes from). Then restart autocue serve.
No tracks shown / "0 tracks"Analyze the library in Rekordbox first (BPM + phrase). AutoCue only sees analyzed tracks.
Port already in useRun on another port: autocue serve --port 8080 (then open localhost:8080).
Windows machineThe DB path isn't auto-detected — pass it explicitly:
autocue serve --db-path "C:\Users\you\AppData\Roaming\Pioneer\rekordbox\master.db"
Python too oldpython3 --version must be ≥ 3.10. Install a newer Python and recreate the .venv.
Something looks wrong after a writeUse Restore backup… in the app, or copy a file from ~/.autocue/backups/ back over master.db (Rekordbox closed).

Updating & removing

  • Update: in the AutoCue folder, git pull then pip install -e . again.
  • Remove: stop the server, delete the AutoCue folder. Backups in ~/.autocue/ can be deleted separately.

Part 2 · Using AutoCue

When AutoCue opens it lands on the Crate Console — a dense, keyboard-friendly workbench. local mode

The layout

  • Left rail — crates (All tracks, No cues yet, Phrase-ready, Already cued), your Rekordbox playlists, saved filters, and a library-health ring.
  • Centre — the track grid (one thin row per track: #, title/artist, BPM, key, energy, mix score, class, cues).
  • Right — the inspector drawer. Click any row and it slides in from the right with that track's energy curve, scores, cues, a ▶ Play button, and a "find similar" action. Click another row to switch focus; it clears when you open a rail place (Library / Duplicates / Discover).
  • Top bar — playlist filter, analysis-mode toggle, Restore backup, Open Nightboard, and a live status sentence.
  • ⌘ K — command palette: jump to any place or action by typing.

Placing hot cues (the core job)

1
Pick tracks. Tick rows in the grid, or use a crate/playlist to scope the set. (No selection = the current filtered set.)
2
Choose a mode in the top bar: Bar intervals (a cue every N bars from bar 1) or ✨ Phrase analysis (cues at detected Intro / Verse / Drop / Bridge / Outro boundaries — needs Rekordbox phrase data).
3
Preview. AutoCue shows the cues it would write as coloured badges on each card, with a confidence level and a reasoning pop-up (the ℹ button).
4
Apply (Rekordbox quit). Cues are written straight into the Rekordbox database after an automatic backup. Re-open Rekordbox to see them.
Cue colours (phrase mode)
Intro = green · Verse = blue · Drop = red · Bridge = cyan · Outro = orange · Build = pink. Slots are ordered A→H with a memory cue (CDJ Auto Cue) first.

Previewing audio

Select a track and press ▶ Play in the inspector drawer to hear it (the original file must still be on disk). A green playhead traces the energy curve as it plays; press again to pause.

Library tools (the "Library" rail place)

ToolWhat it gives you
Health checkA 0–100 library score + a list of issues (missing cues, no beat grid, no phrase analysis, duplicate or unnamed cues…), with one-click fixes. The overall score rolls up when a scan finishes.
Energy & mixabilityPer-track energy curve and a 0–100 mixability score (how easy it is to blend) with an animated breakdown.
ClassificationSorts each track into DJ-set roles (warm-up, build, peak, etc.).
Similar tracksFinds harmonically + energetically close tracks (6-dimension match with a BPM gate).
Transition scoringScores how well any two tracks mix (BPM 40% · key 35% · energy 25%) with advice.
Set builderBeam-search a full set from a start track, target BPM curve, duration, and energy mode.
Cue tools · Discogs · Comments · Playlist suggestBulk cue cleanup, Discogs metadata enrichment, comment/My-Tag enrichment, and category-based playlist suggestions.

Other places & modes

  • Duplicates (rail) — finds duplicate tracks by artist/title/length, picks the best keeper (you can override), and safely deletes the rest with a backup + inline undo.
  • Discover (rail) — surfaces new releases based on the artists and labels you already play (plus a novelty feed); save, dismiss, or snooze.
  • Nightboard (top bar → Open Nightboard) — a full-screen set-building canvas: tiles for tracks, scored "joints" between them, swap/insert alternatives, then export to a playlist.
  • Drag-to-playlist — drag any grid row onto a playlist in the left rail to add it (the row glows green as a valid target; the count pops on drop). Rekordbox must be quit.
  • Restore backup (top bar) — roll back the last database change.
The command-line alternative
Prefer the terminal? autocue --library --dry-run previews cues for the whole library without writing; drop --dry-run to generate a Rekordbox-importable XML instead of writing to the database. The local server in this guide is the recommended path — it is the only mode with the intelligence features above (health, mixability, similar, transitions, set builder, Discover, Nightboard).
(A browser-only XML-in / XML-out web app also exists in the project, but it is not currently hosted at a public URL — run the local server instead.)
