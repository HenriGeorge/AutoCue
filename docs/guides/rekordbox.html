Rekordbox Mastery

Everything you need to organise your library, prepare tracks, and export them to USB for a seamless performance.

Library Management

Importing Tracks

Rekordbox tracks your library by file path. The safest workflow is to keep all music in one stable folder before importing.

  1. Choose a permanent music folder (e.g. ~/Music/DJ Library). Never move files after importing — Rekordbox will lose them.
  2. Drag the folder onto the Rekordbox window, or use File → Add Folder to Collection.
  3. Right-click the Collection and choose Analyse Tracks — this runs BPM detection, beatgrid, and key analysis in one pass.
  4. Enable Preferences → Library → Relocate Missing Files Automatically so Rekordbox can recover if you do move files.
Tip: Analyse before cue-pointing Always let Rekordbox finish analysis before you set cue points. If BPM or beatgrid is wrong, any bar-snapped cues will be in the wrong place.

My Tags & Color Coding

My Tags is Rekordbox's custom tagging system — more powerful than star ratings because you can create any tag you like and filter on them. Find it at the top of the left sidebar.

Energy tags

  • Energy 1 – 5 (peak, after-dark, warm-up…)
  • Opening / Peak / Closing
  • Crowd Pleaser / Deep / Experimental

Function tags

  • Intro-friendly (long intro for mixing in)
  • Acapella Out (vocal ending)
  • Key Locked (avoid harmonic mixing)
  • Re-Edit / Extended

Track color coding (right-click → Color) gives instant visual scanning in your library and on CDJ waveforms:

  • Red = peak / high energy
  • Orange = crowd-pleasers / anthems
  • Yellow = warm-up / cool-down
  • Blue = deep / atmospheric
  • Purple = experimental / risky
Pro tip Agree on a color system before you start and stick to it. Inconsistent tagging is worse than no tagging — you'll second-guess yourself at 2 AM.

Smart Playlists (Intelligent Playlist)

Smart playlists update automatically based on rules — great for "all tracks I haven't played in 6 months" or "everything BPM 125–135 tagged Energy 4–5".

  1. Right-click the Playlist section → Create Intelligent Playlist.
  2. Add conditions: BPM range, My Tag, genre, date added, play count, rating, key, etc.
  3. Set Match: All / Any for AND vs OR logic.
  4. The playlist rebuilds every time you open it — no manual maintenance.

Useful preset recipes:

  • New additions: Date Added is in the last 30 days
  • Peak set candidates: BPM 128–135, My Tag contains "Energy 5"
  • Needs prep: Has no hot cues (Cue count is 0)
  • Not recently played: Last played is more than 90 days ago

Track Analysis

Rekordbox waveform anatomy — phrase regions, beatgrid, hot cues, memory cues, and playhead

Fixing Beatgrids

A correct beatgrid is the foundation of everything — loops, hot cues, sync, and AutoCue phrase detection all depend on it. Most tracks auto-analyze correctly; here's how to fix the ones that don't.

Check these tracks manually Live recordings, old vinyl rips, jazz, and tracks with tempo variation (any track not produced in a DAW to a click track) will often have incorrect beatgrids.

The beatgrid editing workflow:

  1. Load the track into the deck. In the waveform view, zoom in to the first kick drum.
  2. Press Edit Grid in the Beatgrid panel. A grid overlay appears on the waveform.
  3. If the BPM is wrong: tap Tap BPM along with the beat, or type the correct BPM directly.
  4. If the phase is off (grid is right BPM but misaligned): drag the grid left/right, or use the half-beat shift buttons.
  5. For tracks with genuine tempo variation: disable Quantize in Preferences and set the grid to Dynamic mode — this lets Rekordbox follow the tempo changes instead of locking to a single BPM.
  6. Press Save. The corrected grid is stored in Rekordbox and will appear on CDJs when you export to USB.
Dynamic vs Static analysis Static (default) locks the entire track to one BPM. Dynamic follows tempo variation. Use Dynamic for live recordings, jazz, and anything with a human drummer. Use Static for electronic music produced to a click track.

Phrase Analysis

Phrase analysis (available in Rekordbox 6+) detects structural sections of a track — Intro, Verse, Chorus, Bridge, Outro — and stores them in the ANLZ binary analysis files on disk. This is what AutoCue's phrase mode reads.

  • Phrase analysis runs automatically as part of Analyse Tracks.
  • The coloured phrase strip appears above the waveform in Performance mode.
  • Rekordbox labels phrases by mood (1 = mid-energy, 2 = high-energy, 3 = low-energy) and kind (numeric 1–10 mapping to Intro/Verse/Chorus etc.).
  • Intelligent Cue (Creative/Professional Plan only) can auto-place hot cues at phrase boundaries. AutoCue replicates this for free in both the web app and CLI.

Cue Points & Loops

Hot Cues

Hot cues are instant-jump markers assigned to buttons A–H. They trigger immediately when pressed — no need to pause — making them your primary performance tools.

When to set them

  • A — Track start / intro beat 1
  • B — First drop / main hook
  • C — Breakdown
  • D — Build-up
  • E — Second drop
  • F–H — Outro / acapella / special moments

Naming convention

Rekordbox displays cue names on CDJ screens. Short names are best:

  • Intro / Drop / Build / Break / Outro
  • Vocal / Acap / Inst
  • Mix In / Mix Out (for long mixes)

AutoCue uses: Intro, Verse, Chorus, Bridge, Outro, Up, Down

Color coding hot cues speeds up visual recognition on the CDJ touchscreen:

  • Green = safe mix-in point
  • Blue = mix-out / outro
  • Orange = builds / drops
  • Red = high energy / peak moment
  • Pink = vocals / acapella
Auto-load hot cues on CDJ On the CDJ-2000NXS2 and CDJ-3000: press MENU → Utility → Auto Cue → set to Memory. This loads the first memory cue automatically, jumping to your prepared start point rather than the very beginning of the file. Hot cues always load regardless.

Memory Cues

Memory cues are navigation markers. They don't trigger on button press — they show as triangles on the waveform. Use them for:

  • Marking where a long intro starts getting interesting
  • Flagging a tricky timing section
  • Recording mix-in / mix-out points without using up hot cue slots

The first memory cue is special — it sets the Auto Cue position. When you load a track on a CDJ with Auto Cue enabled, playback begins at the first memory cue, not time 00:00.

Loops & Beat Jump

Loops let you extend a section or create tension. Beat Jump moves the playhead exactly N beats forward/back without breaking the beatgrid.

Setting loops

  • Auto Beat Loop: click a loop size (1/2 beat, 1, 2, 4, 8 bars) and it starts immediately. Best for live performance.
  • Manual loop: press Loop In at the start, Loop Out at the end. Fine-tune using the jog wheel while paused — in Rekordbox the jog adjusts loop endpoints with ±sample precision.
  • Saved loops: right-click any loop → Save as Hot Cue Loop. The loop is stored in a hot cue slot (shown in orange on CDJs).
  • Loop Halve / Double: instantly cuts or doubles the active loop length — great for building tension.

Beat Jump

Beat Jump skips the playhead forward or backward by an exact number of beats while keeping sync. Use it to:

  • Jump to the chorus 32 bars early without losing beatmatch
  • Correct a late mix by jumping forward 4 beats
  • Navigate to specific structural points quickly in preparation mode

Performance Mode Tips

Sync & Quantize

Sync locks BPM and phase between decks. Quantize snaps cue triggers and loop points to the nearest beat so they always fire on the grid. Both are tools, not cheats — even experienced DJs use them strategically.

  • Enable Quantize in Preferences → Controller → General to snap hot cue presses to the beat.
  • Turn Quantize off for free-form scratching or off-beat fills — you want immediate trigger response.

FX in Rekordbox

Rekordbox's built-in FX (in Performance mode with a compatible controller) mirror the DJM-900NXS2 effects section:

  • Filter: most-used effect. HPF sweeps mud out of a mix; LPF adds warmth and hides a rough transition.
  • Echo: use on the outgoing track's last phrase to extend its reverb tail into the incoming track.
  • Beat FX Beat: always set beat multiplier to match the track's phrasing (×2 or ×4 for most electronic music).

Interface changes

  • Icon-based Media Browser: replaces the old tree/folder view with a visual, icon-driven layout. Navigation is more consistent with CDJ system menus.
  • Column View: hierarchical browse similar to macOS Finder — navigate folder-to-folder without losing context.

  • Dark and Light modes: both fully overhauled with better waveform contrast and control grouping.

AI features (Creative/Professional plan)

  • Collection Radar: analyses your library usage patterns and surfaces tracks you haven't played recently but that fit the current moment. Think "Spotify Discover Weekly" but for your own library.
  • Streaming Radar: searches Beatport, SoundCloud and other connected streaming services to suggest new tracks based on your current set's mood, BPM, and key.

Workflow tools

  • Dual Player Layout (Export mode): links two decks for synchronized playback with Beat Jump, EQ Kill, Key Sync, and Key Shift — useful for pre-listening to transitions before a gig.
  • Collaborative Playlists: share playlists with other Rekordbox 7 users while preserving BPM grids and cue point metadata.

USB Export

Exporting to USB incorrectly is one of the most common causes of "the CDJ doesn't see my tracks" at the venue.

Format your USB correctly first

Always use FAT32 + MBR — not exFAT, not APFS, not GPT CDJ-2000NXS2 and older CDJs require FAT32 with an MBR (Master Boot Record) partition table. CDJ-3000 supports exFAT, but FAT32+MBR is universally compatible with all Pioneer gear. For drives >32 GB on macOS, use Disk Utility → Erase → MS-DOS (FAT) + Master Boot Record scheme.
  1. Open Rekordbox. Connect the USB drive. It appears under Devices in the left sidebar.
  2. Drag playlists from your Collection onto the device — or use Export → Sync Manager for a rules-based sync.
  3. Use Sync Manager (recommended): define which playlists sync to which device. Rekordbox handles additions and deletions automatically.
  4. Do not manually drag music files onto the USB in Finder/Explorer. Rekordbox needs to write the PIONEER/ database folder alongside your music.
  5. Always eject safely — the CDJ reads a database that can corrupt if the drive is yanked mid-write.

Pre-export checklist

  • All tracks analysed (no orange waveforms)
  • All beatgrids verified for tracks you plan to sync on
  • Hot cues set on key tracks
  • USB formatted FAT32 + MBR
  • Eject safely after export
  • Test the USB on a CDJ before the gig (at home or at the venue early)
Backup protocol Keep at least 2 USBs. Format and sync both before every gig. CDJs at clubs can corrupt drives. Carry a spare. Professional DJs often carry 3.

USB Drive Selection Guide

Not all USB drives are equal. Slow or unreliable drives cause track loading hangs on CDJs. Here are verified options ordered by performance:

DriveRead SpeedFormatNotes
SanDisk DJ Flash Drive (2025) NEW ~1,000 MB/s FAT32 / exFAT Pioneer/AlphaTheta co-engineered; dual USB-A + USB-C; 512 GB – 1 TB. Best choice for CDJ-3000 setups.
Kingston DataTraveler Max up to 1,000 MB/s FAT32 / exFAT USB 3.2 Gen 2. Solid long-term reliability. Widely available.
SanDisk Extreme PRO up to 420 MB/s FAT32 / exFAT Rugged aluminium body. Reliable for touring. Good for mixed CDJ generations.
SanDisk Ultra Fit 130–150 MB/s FAT32 / exFAT Budget pick. Adequate for MP3/AAC sets. Not ideal for lossless AIFF/WAV-heavy libraries.

Minimum specs: USB 3.0, sustained read speed ≥ 80 MB/s. The CDJ-3000 buffers full tracks into RAM on load, so only read speed matters — write speed is irrelevant. 64–128 GB is sufficient for most gigs; a 3-hour AIFF set is roughly 5–7 GB.

Avoid no-name drives Unbranded USB drives often have inconsistent read speeds due to low-quality NAND flash. A drive that reads fine on your computer may stall mid-track on a CDJ. Stick to SanDisk, Kingston, Samsung, or Corsair.

Advanced Techniques

Advanced Library Architecture

Once your library grows past a few hundred tracks, the default flat collection view stops working. Professional DJs use a deliberate folder and playlist hierarchy built around how they actually select tracks at a gig, not how they organise music at home.

The Preparation Folder workflow

Create a special playlist called _PREP (underscore keeps it at the top). Drag every track into it when you import it. This is your "inbox" — tracks that haven't been analysed, tagged, or cue-pointed yet.

  1. Import new tracks → they go into _PREP.
  2. Work through _PREP: fix beatgrids, set hot cues, add My Tags, set key.
  3. Once a track is fully prepared, remove it from _PREP and add it to its genre/energy playlists.
  4. An empty _PREP folder means your library is fully prepared. A full one means you have homework.

Nested playlist structure

Rekordbox supports playlist folders (right-click → Create Folder). A proven structure:

By energy arc

  • 📁 Warm-up (BPM 118–124, energy 1–2)
  • 📁 Build (BPM 124–128, energy 3)
  • 📁 Peak (BPM 128–133, energy 4–5)
  • 📁 After-dark (dark, minimal, late night)
  • 📁 Closing (slower, emotional, outro-ready)

By function

  • 📁 CURRENT SETS (active playlists for gigs)
  • 📁 HISTORY (auto-generated play history)
  • 📁 _PREP (unfinished tracks)
  • 📁 TOOLS (acapellas, intros, anthems)
  • 📁 ARCHIVE (old but keep-worthy)

History playlists

Rekordbox automatically records every track you play in History (bottom of the left panel). After each gig, right-click the session → Export to Playlist. Name it with the date and venue. Over time this becomes invaluable: you can see exactly which tracks you've overplayed, which you've never reached for, and which combos worked.

Smart Playlist: tracks not played in 6 months Create an Intelligent Playlist with condition "Last Played more than 180 days ago" + "My Tag does not contain Archive". This surfaces forgotten gems that are still prepared and ready.

XML as a Power Tool

Rekordbox's File → Export Collection in rekordbox format writes your entire library — tracks, playlists, hot cues, beatgrids, My Tags — to a human-readable XML file. This is more powerful than it looks.

Library migration and backup

  • Full backup: export XML monthly. If Rekordbox's master.db corrupts (it happens), you can rebuild from XML in minutes rather than re-preparing every track from scratch.
  • Moving computers: export XML on old machine, import on new one. All metadata survives — hot cues, tags, playlists, beatgrids.
  • Sharing prep work: send an XML to a b2b partner so they start the gig with your cues already loaded.

Third-party tools that read Rekordbox XML

  • Lexicon DJ: migrate libraries between DJ software (Rekordbox ↔ Serato ↔ Traktor ↔ Denon Engine). Reads and writes Rekordbox XML. The most reliable cross-platform migration tool.
  • DJ Data Converter: free, open-source XML conversion for basic migrations.
  • AutoCue: reads your Rekordbox XML and writes back hot cues at phrase boundaries. AutoCue's output is a Rekordbox XML — import it via File → Import Library.
File → Import Library vs Export Collection These are different directions. Export Collection writes your full library to XML. Import Library merges an XML back in — it only writes data that's present in the XML; missing tracks and slots are left untouched. AutoCue uses Import Library specifically because it's additive, not destructive.

Editing XML directly (power users)

The Rekordbox XML format is documented and editable. Advanced users write scripts to bulk-set track colours, rename cues programmatically, or normalise My Tags across 10,000 tracks. The <TRACK> element holds everything; <POSITION_MARK> elements are hot/memory cues. The pyrekordbox library (Python) gives full programmatic access.

Streaming Integration (SoundCloud / Beatport Link)

Rekordbox integrates with streaming services on the Creative and Professional plans. You can browse and play streaming tracks directly from the Rekordbox player interface and CDJ screens.

  • SoundCloud DJ: access SoundCloud's catalog from Rekordbox. Requires a paid SoundCloud DJ subscription in addition to a Rekordbox Creative/Pro plan.
  • Beatport Link: access the full Beatport catalog for streaming and offline caching. Industry-standard for keeping up with new releases before buying.
  • Beatsource Link: similar to Beatport, focused on open-format DJing (hip-hop, R&B, dance).
Streaming limitations at venues Streaming tracks require an internet connection to play. Club Wi-Fi is unreliable. Most professional DJs treat streaming as a discovery tool, then download and import tracks they plan to gig with. Do not rely on streaming for a paid gig unless you have a verified reliable connection and offline caching enabled.

Offline mode / Locker: tracks you've played from a streaming service can be cached locally ("added to Locker") for offline use, subject to the service's terms. Cache your set tracks the day before.

Large Library Management (10,000+ Tracks)

Libraries above ~5,000 tracks introduce performance and navigation challenges. Here's how professional DJs with large collections stay fast.

Search discipline

  • Use Ctrl/Cmd+F to focus the search bar instantly. Rekordbox searches across title, artist, album, and remixer simultaneously.
  • Prefix searches: type t:word to search title only, a:artist for artist only — faster on large databases.
  • The Tag List sidebar filter (show/hide with View → Tag List) lets you quickly filter by genre, key, or BPM range without building a playlist.

Duplicate detection

Rekordbox has no built-in duplicate finder. Use Mixed In Key or Duplicate Cleaner Pro (Windows) / dupeGuru Music (Mac) to find audio duplicates by waveform fingerprint rather than filename. Re-import cleanly after removing duplicates.

Database maintenance

  • Relocate Missing Files: right-click Collection → Relocate Missing Files. Run this whenever files have moved. Better: never move files after importing.
  • Remove from Collection (not just playlist): when permanently deleting tracks, right-click → Remove from Collection, not just Delete from Playlist. Orphaned collection entries slow down the database over time.
  • Database backup: the master database lives at ~/Library/Caches/rekordbox/ (macOS) or %APPDATA%\Pioneer\rekordbox\ (Windows). Back it up alongside your XML exports. Time Machine on macOS does this automatically if configured.

BPM and key corrections at scale

For bulk-correcting analysis errors (common after a large import), use Preferences → Analysis to re-analyse selected tracks with updated sensitivity settings, rather than fixing each one manually. Set the BPM range to match your genre (e.g. 120–140 for techno/house) to prevent half/double BPM detection errors.

12 Common Rekordbox Mistakes

  • Moving files after importing. Rekordbox tracks absolute paths. Moving or renaming files breaks library links — always import from their final location.
  • Not analysing before setting cues. If the BPM is wrong, all bar-snapped cues will be misaligned.
  • Trusting auto-analysis on live recordings. Manually verify beatgrids for any track with a real drummer or tempo variation.
  • Using exFAT USB for old CDJs. CDJ-2000NXS2 and below don't read exFAT. Always use FAT32+MBR.
  • Manually copying files to USB in Finder. Rekordbox writes a database; you can't bypass it. Use the app's export.
  • Forgetting to eject the USB safely. A corrupt database means no cues, no analysis on CDJs.
  • One USB, no backup. Drives fail. Always have two.
  • Skipping key analysis. Rekordbox detects key automatically. Without it, you can't use harmonic mixing features.
  • Setting cues mid-analysis. Wait for the orange waveform to turn blue/grey before setting cues — analysis can shift timing.
  • Ignoring the first memory cue. It sets the Auto Cue position on CDJs. If it's at 00:00, your tracks always load at the very start including silent intros.
  • Never cleaning the collection. Duplicate tracks, missing files, and stale playlists slow down library navigation at 3 AM. Schedule a monthly clean-up.
  • Over-relying on sync without understanding phrasing. Sync matches BPM but not musical phrases. If you mix in at the wrong bar, it'll still sound wrong even if the beats line up.
