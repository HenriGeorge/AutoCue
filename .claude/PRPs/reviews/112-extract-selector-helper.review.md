## Audit

- **Correctness:** `buildIdSelector` is moved unchanged. The spec
  imports the same symbol from a sibling module. `per-control-sweep.helpers.ts`
  contains zero `test()` calls — Playwright will not interpret it as
  a test file under the default `*.{spec,test}.{ts,js}` glob, so
  discovery sees the per-control rows again.
- **Security / safety:** No write-path code touched. No CORS change.
  No master.db code touched. Sandbox preflight (`safety.spec.ts`)
  now runs through the documented entry point again — the audit
  surface that #112 disabled is restored.
- **Test quality:**
  - Regression for #20 unchanged (would fail if `CSS.escape` returned).
  - Regression for #112 added in three layers:
    1. `selector.test.ts` source must not import a sibling `.spec/.test`.
    2. `per-control-sweep.spec.ts` source must not import a sibling
       `.spec/.test`.
    3. Self-check that the guard regex catches the forbidden pattern
       (`from "./per-control-sweep.spec"`) and accepts the safe
       sibling helper (`from "./per-control-sweep.helpers"`).
    Without (3) a future regex regression could silently disable the
    guard with all assertions still passing.
  - Property assertion in (3) (`toBe(true)` / `toBe(false)` on a
    constructed shape) rather than specific-value scoring.
- **Patterns:** Helper file follows the same module-comment convention
  as `control-inventory.ts` (sibling pure-data module also imported
  by spec files). `package.json` and `playwright.config.ts` are
  unchanged — no shared-roots fan-out trigger.
- **Types:** `buildIdSelector(id: string): string` unchanged. New
  regression block uses `node:fs`, `node:url`, `node:path` — all
  stdlib, already available under Node 18+.
- **Scope:** No drive-by refactors. Issue body's suggested fix
  implemented as-is, plus one extra layer of regression guard
  beyond the brief because issue #112 had a CRITICAL severity label
  and the cost of a recurrence is the entire safety harness silently
  going offline again.
