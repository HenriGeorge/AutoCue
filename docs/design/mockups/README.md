# AutoCue 2.0 — design mockups

Static HTML mockups from the redesign exercise (2026-06-12). Open any file in a
browser. Each has a working light/dark toggle and wired key interactions.

| File | Concept | Role |
|---|---|---|
| `design-A.html` | **Crate Zero** | Radical minimalism — ⌘K palette + status sentence + one ink pill |
| `design-B.html` | **Crate Console** | Pro workbench — rail + dense grid + inspector. **The chosen home.** |
| `design-C.html` | **Pre-Flight Dial** | Health-ring homepage + prioritized fix stack |
| `design-D.html` | **Nightboard** | Spatial set timeline — energy-curve tiles + scored transition joints |
| `design-E.html` | **Synthesis** | First A+B+C+D synthesis (rail modes + global palette/dock) |
| `design-F.html` | **Booth Partner** | AI-native conversational console with live artifacts |
| `design-G.html` | **The AutoCue Review** | Editorial data-magazine — library as a typeset issue |
| `design-H.html` | **Stagehand** | Ambient zero-UI — menubar consent panel + weekly digest |
| `design-Z-endstate.html` | **End State** | **The canonical P2 target** — workbench fully realized: top-bar toolbar, rail (crates/playlists/saved-filters/health-ring), dense grid with F proposal stamps, rich inspector, anchor-transition card |

**Locked direction** (socratic grill, see `.claude/PRPs/prds/autocue-2-program.prd.md`):
B "Crate Console" is home; the build chases `design-Z-endstate.html`. F/G/H donated
organs (F proposal→applied stamps + approve ticks; G deterministic lede; H

door (P6), seeded by the ⌘K composer seam.
