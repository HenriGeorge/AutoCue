Run the `autocue-qa` agent. It boots `autocue serve` against a sandbox copy of master.db (verified via `/api/status` + `X-AutoCue-Diagnostic: 1`), runs the Playwright smoke suite (safety + selectors + API + SSE + UI + Pages mode), then drives Chrome DevTools through the **documented feature sweep** — every user-facing flow in `docs/reference/` (cue-generation, library-health, comment-enrichment, cue-library-tools, set-builder, similar-tracks, transition-scoring, playlist-suggest, auto-tag, discogs-and-discovery, youtube-download). Files GitHub issues for failures with fingerprints keyed to the reference doc. Issue filing requires one-time per-repo consent and is capped at 10 issues per run; pass `--dry-run` to write the report only. Summary lands in `.claude/reports/autocue-qa-<date>.md`.
