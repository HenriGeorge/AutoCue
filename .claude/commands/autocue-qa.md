Run the `autocue-qa` agent. It boots `autocue serve` against a sandbox copy of master.db (verified via `/api/status` + `X-AutoCue-Diagnostic: 1`), runs the Playwright smoke suite (safety + selectors + API + SSE + UI + Pages mode), and files GitHub issues for every failure found. Issue filing requires a one-time consent step per repo and is capped at 10 issues per run; pass `--dry-run` to write the report only. Writes a summary to `.claude/reports/autocue-qa-<date>.md`.
